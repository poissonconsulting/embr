## Package Overview

`embr` is a foundational R package providing shared utility functions and virtual S3 classes for the "model builder" family of Bayesian modeling packages (`jmbr` for JAGS, `smbr` for Stan, `tmbr` for TMB). It defines the common API and abstractions; backend packages implement the engine-specific methods.

## Architecture

### S3 Virtual Class Hierarchy

`embr` defines generic S3 classes that backend packages extend with engine-specific methods:

- `mb_code` ŌĆö model code representation
- `mb_model` ŌĆö model definition: code, priors, data selection, transformations
- `mb_analysis` ŌĆö results from a single analysis run
- `mb_analyses` ŌĆö collection of analysis results
- `mb_null_analysis` ŌĆö null/baseline comparison object

The key pattern: `embr` defines generics and common behavior; `jmbr`/`smbr`/`tmbr` implement `analyse.jmbr_model`, `analyse.smbr_model`, etc.

### Analysis Workflow

1. User creates a model via `mb_model()` with code, priors, data selection, and `modify_data()`/`new_expr()` transformations
2. `analyse()` dispatches to the engine-specific method, which fits the model and returns an `mb_analysis` object
3. Results are extracted via `broom` generics (`coef()`, `glance()`, `tidy()`, `augment()`) plus package-specific functions (`IC()`, `R2()`, `sensitivity()`, `estimates()`)

### Data Pipeline

Models carry a data transformation pipeline:
- `select_data()` ŌĆö subset columns/rows before model fit
- `modify_data()` ŌĆö transform data (e.g., scale variables)
- `new_expr()` ŌĆö define derived parameters calculated post-fit using `mcmcderive`

### Key Files

- `R/model.R`, `R/base-model.R` ŌĆö model construction
- `R/analyse.R`, `R/analyses.R` ŌĆö generic analysis functions
- `R/coef.R`, `R/IC.R`, `R/R2.R`, `R/sensitivity.R` ŌĆö results extraction
- `R/powerscale-sensitivity.R`, `R/priorsense-helpers.R` ŌĆö sensitivity analysis via `priorsense`
- `R/pmb-analyse.R` ŌĆö PMBR (partial model builder regex) support
- `R/backwards.R` ŌĆö backwards compatibility shims

### Test Objects

Integration tests use pre-fit RDS objects in `inst/test-objects/` to avoid slow MCMC fitting in CI. These are regenerated by `inst/create-test-analyses.R`. **Regenerating them requires updating snapshots** ŌĆö see the warning at the top of that script.

Tests prefixed `zzz-` (e.g., `test-zzz-analyse.R`) run last and require `jmbr` to be installed.

### Global Options

The package uses `options()` for defaults (e.g., `mb.nchains`, `mb.niters`). These are accessed via `get_analysis_mode()`/`set_analysis_mode()`.

### Dependencies

Key upstream packages (many from poissonconsulting GitHub):
- `mcmcr`, `mcmcdata`, `mcmcderive` ŌĆö MCMC results handling
- `posterior`, `priorsense` ŌĆö posterior analysis and sensitivity
- `universals`, `extras`, `chk` ŌĆö shared utilities
- `newdata` ŌĆö prediction data generation
